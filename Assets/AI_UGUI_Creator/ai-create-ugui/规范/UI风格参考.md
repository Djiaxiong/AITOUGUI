# UI风格参考

这份文档用于扩充 `draft_visual_intent.md` 的风格判断和提示词信息密度。
它不是“固定模板库”，也不是“照抄就能出图”的页面壳。

目标只有两个：

1. 帮 AI 把题材、材质、轮廓、信息节奏说清楚
2. 帮同类游戏 UI 拉开足够大的风格差异，避免只换配色

## 一、先明确什么叫“风格”

游戏 UI 风格不能只写成：

- 仙侠
- 赛博朋克
- 蒸汽波
- 蓝金高级感

这类词太宽，最后很容易收成同一套骨架。

一份合格的风格意图，至少要同时回答下面 8 个问题：

1. 这个世界的材质来自哪里：金属、玉石、纸张、漆木、皮革、石头、塑料、能量屏？
2. 页面骨架像什么：舞台、驾驶舱、仪表盘、卷轴、指挥台、档案夹、祭坛、工坊？
3. 轮廓是硬的还是软的：直角、圆角、切角、旗帜形、牌匾形、法阵形？
4. 边框语言是什么：实边、细线、描金、发光、铆钉、压印、虚线？
5. 装饰来自哪里：徽章、封蜡、卡扣、刻线、法纹、线路、布标、缝线、贴纸？
6. 空间感怎么建立：压印、内凹、投影、背光、悬浮、层叠、透片？
7. 字体语气是什么：英勇、技术、祭仪、童趣、档案、军令、演出？
8. 交互反馈像什么：机械按压、霓虹脉冲、纸片翻页、法阵呼吸、玩具弹跳？

## 二、反趋同硬规则

1. 不允许只换颜色，不换骨架。
2. 同类题材至少要在 `layout / shape / frame / ornament` 四轴里拉开 3 轴。
3. 先决定材质和信息组织，再决定具体颜色。
4. 低置信度时先减装饰，不要直接退回“通用蓝金手游面板”。
5. 写提示时要主动写禁止项，明确不能落成什么。

常见失败收口：

- 通用蓝金手游
- 通用赛博黑底霓虹
- 通用电商首页分栏
- 通用后台仪表盘
- 通用奶油圆角卡片

## 四、风格族参考

下面这些不是模板，而是可写入提示的设计方向。
每次只选一条主路线，不要把多条互相冲突的路线混成一锅。

### 1. 军事指挥 / 战区推演

- 适合：RTS、塔防、现代战争、战棋、指挥模拟
- 视觉提示：地图台、标绘线、坐标框、军令标签、侧边命令板、压暗背景、低饱和底色上点亮指令色
- 轮廓建议：偏直角、切角、小半径圆角、硬质矩形
- 边框建议：细线、刻度线、双层描边、虚线框
- 色彩关系：大面积灰绿/枪灰/煤黑，强调色只给警报、选中、攻击、补给
- 组件提示：按钮像命令牌，卡片像情报条，面板像折叠指挥册
- 避免：大面积玻璃拟态、豪华金边、娱乐化彩色糖果按钮

### 2. 工业重机 / 机库作业

- 适合：机甲、装甲改装、载具管理、基地建造、科幻维修
- 视觉提示：装甲板拼缝、铆钉、编号、黄黑警示、结构梁、检修贴签、模块舱位
- 轮廓建议：切角板、厚边矩形、梯形感标题条
- 边框建议：厚描边、内凹压边、结构接缝、警示条
- 色彩关系：深灰/铁黑做主底，黄橙做风险与提示，少量冷白或蓝白做信息照明
- 组件提示：背包像仓位，升级卡像设备铭牌，主 CTA 像机械锁扣按钮
- 避免：漂浮轻盈、过多圆润果冻边、纯霓虹荧光主导

### 3. 蒸汽航海 / 黄铜导航

- 适合：航海经营、蒸汽冒险、贸易、舰船管理、空艇题材
- 视觉提示：黄铜仪表、表盘刻线、罗盘环、皮革压纹、铆钉、航海图、绳结、木框
- 轮廓建议：圆角牌匾、圆环、半圆仪表、舱门式面板
- 边框建议：金属包边、铆钉边、细刻线、双层环框
- 色彩关系：深木、煤黑、黄铜、旧纸白做主系，海蓝或赤铜做少量强调
- 组件提示：页签像航海日志，数值模块像表盘，标签像货运签章
- 避免：大面积未来感纯黑霓虹、现代扁平后台表格

### 4. 黑暗奇幻 / 哥特祭坛

- 适合：暗黑 RPG、吸血鬼、恶魔城、地牢、宗教审判题材
- 视觉提示：石板、铁链、尖拱、烛光、封印、浮雕、污金、深红丝绒
- 轮廓建议：高挑竖向框、尖角牌、祭坛式中轴
- 边框建议：冷铁边、浮雕边、破损线、封印环
- 色彩关系：近黑底色配酒红、骨白、旧银、暗紫灰；高亮只留给圣火或诅咒色
- 组件提示：主菜单像祭坛，人物面板像圣龛，技能框像封印石匣
- 避免：明快可爱圆角、金碧辉煌的王国气质、轻社交 App 感

### 5. 王国史诗 / 高幻想徽章

- 适合：MMORPG、王国经营、骑士冒险、战役地图、英雄养成
- 视觉提示：徽章、绶带、王冠、盾牌、纹章、锦缎、雕花金边、宝石扣件
- 轮廓建议：旗帜、牌匾、盾形、长卷轴式面板
- 边框建议：描金、丝带边、花纹边、勋章角饰
- 色彩关系：底色压深，主亮色集中在金、宝石蓝、象牙白、绯红强调
- 组件提示：CTA 像御令按钮，角色卡像勋章板，页签像书册索引
- 避免：整屏蓝金铺满、每个模块都同样豪华、失去层级重点

### 6. 仙侠修真 / 玉石法阵

- 适合：修仙、门派、洞府、法宝、御剑、炼丹、国风奇幻
- 视觉提示：玉石渐层、墨青、云纹、题签、印章、法阵环、卷轴、悬浮符箓
- 轮廓建议：牌匾、法阵圆环、题签条、柔和不对称圆角
- 边框建议：细金边、玉质内描、印纹角饰、环形描线
- 色彩关系：主底宜静，不要全屏高亮；青玉负责过渡，月白负责留白，墨青负责压重，鎏金只做锚点
- 组件提示：背包格可像锦囊盒，角色页像洞府案台，主 CTA 像题签或灵符
- 避免：全屏国潮电商红金、廉价蓝金手游壳、过量荧光霓虹

### 7. 武侠江湖 / 水墨卷宗

- 适合：武侠、门派经营、江湖叙事、探案、古风策略
- 视觉提示：纸张纤维、墨色层次、印泥、装订线、木夹、账册、竹简、笔触留白
- 轮廓建议：册页、条幅、纸签、案卷分栏
- 边框建议：细墨线、印章压角、纸边、绢布边
- 色彩关系：纸白、墨灰、褐木色为主体，红印和靛青只作点睛
- 组件提示：日志像卷宗，任务卡像悬赏纸，页签像书签和纸签
- 避免：大面积厚重金属边、每块都发光、完全现代化 HUD

### 8. 赛博终端 / 黑客渗透

- 适合：黑客、网络攻防、潜行、未来谍战、赛博动作
- 视觉提示：黑底终端、扫描线、矩阵格、细切角、高对比 HUD、信号中断、数据流
- 轮廓建议：锐角切边、窄边框、长条状态模块
- 边框建议：发光细线、分段框、断裂边、示波器风描线
- 色彩关系：大面积近黑，青绿或电蓝做主提示，洋红或橙红只给风险或高优先级交互
- 组件提示：卡片像终端窗口，按钮像命令片，信息区像侦测读数
- 避免：通用玻璃拟态、厚重拟物金边、过多柔圆卡片

### 9. 硬科幻舰桥 / 战术终端

- 适合：太空舰桥、硬科幻、舰队指挥、机库总控、战术部署
- 视觉提示：舰桥面板、雷达环、坐标分区、冷白背光、低饱和深蓝灰、工程标号
- 轮廓建议：舱壁矩形、环形仪表、包裹式控制台
- 边框建议：细刻度、冷白边、结构凹槽、硬投影
- 色彩关系：蓝灰和炭黑压底，白蓝做主要信息光，红橙只给告警
- 组件提示：顶部像舰桥横向状态条，中心像战术屏，侧栏像分系统面板
- 避免：赛博霓虹夜店感、奇幻纹样、柔软手作装饰

### 10. 废土拾荒 / 拼装生存

- 适合：废土生存、拾荒、沙盒建造、幸存者、末日基地
- 视觉提示：补丁、胶带、旧标签、喷漆字、锈迹、拼焊、帆布、废金属
- 轮廓建议：不完全对称的拼接板、旧牌、贴条、粗糙卡片
- 边框建议：破损边、拼缝、虚线缝线、粗标签条
- 色彩关系：土灰、铁锈、尘黄、旧绿作主基底，强调色用信号橙或安全蓝
- 组件提示：物资卡像封装包，任务卡像手写公告，按钮像拼装铁牌
- 避免：过度干净、玻璃光滑、整屏统一精致圆角

### 11. 沙漠遗迹 / 远行商队

- 适合：遗迹冒险、商队经营、沙漠探索、神庙题材
- 视觉提示：砂岩、陶片、织物、护符、图腾、月相、镶嵌、风蚀纹理
- 轮廓建议：拱形、碑牌、护符框、宽扁卡片
- 边框建议：石刻边、织带边、镶嵌线、符纹刻槽
- 色彩关系：赭石、沙金、青绿氧化色、夜蓝阴影，亮色要少而稳
- 组件提示：页签像陶牌，奖励卡像护符牌，选项像商队标签
- 避免：冰冷科技感、荧光霓虹、统一工业硬边

### 12. 卡牌酒馆 / 桌游匣盒

- 适合：卡牌、桌游、肉鸽构筑、酒馆经营、派对博弈
- 视觉提示：木盒、绒布、烫金、筹码、卡背、插槽、票券、规则板
- 轮廓建议：卡片堆、票券、条签、盒盖式面板
- 边框建议：烫金边、纸牌边、磨损边、细压印
- 色彩关系：深木色或酒红压底，象牙白与金色做信息区，强调色收在稀有度和胜负
- 组件提示：按钮像票券，卡片像牌桌实体卡，页签像盒内分隔签
- 避免：移动端电商宫格、过度科技 HUD、全屏发光线条

### 13. 童趣手作 / 玩具拼贴

- 适合：休闲、儿童、轻冒险、幸存者、模拟经营、派对
- 视觉提示：手绘线、彩纸、泡泡、贴纸、蜡笔、木头玩具、布艺、海报感
- 轮廓建议：圆润、歪斜、手工剪裁、贴纸角
- 边框建议：粗描边、纸边、缝线、阴影偏移
- 色彩关系：主底保持明快但不要荧光过量，强调色分清安全、奖励、危险、互动
- 组件提示：按钮像糖纸或木牌，卡片像剪贴纸，面板像手工公告板
- 避免：拟真军武、厚重金属、过度成人化冷黑终端

### 14. 偶像音游 / 舞台节拍

- 适合：音游、偶像养成、演出经营、节奏动作、直播舞台
- 视觉提示：舞台灯条、谱面轨、票根、灯牌、镭射、演出票、闪粉、舞台帘幕
- 轮廓建议：舞台牌、灯框、细长状态条、节拍切片
- 边框建议：霓虹边、票券齿边、高亮细线、舞台框灯
- 色彩关系：主色要有演出主调，但高亮色只留给连击、命中、主 CTA、舞台反馈
- 组件提示：数值像演出票号，页签像曲目标签，卡片像演出海报或应援卡
- 避免：军事仪表盘、低饱和行政后台、厚重中世纪浮雕

## 五、按游戏类型选信息骨架

### RTS / 塔防 / 战术

- 优先强调：主战区、资源条、出兵入口、波次信息、科技切页
- 常见骨架：`dashboard`、`stage + side rails`
- 关键不是豪华装饰，而是命令优先级和读图速度

### RPG / MMO / 主城养成

- 优先强调：角色核心区、成长入口、任务区、活动区、社交区
- 常见骨架：`hero-focus`、`dashboard`
- 关键是中心视觉锚点明确，不要满屏同权重入口

### 生存 / 废土 / 基地

- 优先强调：资源状态、制造、地图、队伍、风险提示
- 常见骨架：`binder`、`dashboard`
- 关键是功能感和材料感要一致，不要把每块都做成节庆海报

### 卡牌 / 构筑 / 桌游

- 优先强调：卡组区、手牌区、规则提示、资源槽、奖励选择
- 常见骨架：`tabletop spread`、`album-grid`
- 关键是实体感、分区感和牌面层级

### 音游 / 演出 / 社交休闲

- 优先强调：节拍区、演出状态、曲库入口、奖励反馈、角色立绘
- 常见骨架：`stage`、`hero-focus`
- 关键是节奏和氛围，不要掉成普通直播后台

## 六、写进提示时建议明确的信息

每次写需求或扩写需求时，至少要带上这些信息：

- 题材和玩家视角
- 页面列表与页面优先级
- 主要材质来源
- 页面骨架像什么
- 轮廓和边框语言
- 颜色关系和亮点投放位置
- 组件家族气质
- 明确禁止项
- 与常见默认风格至少 3 轴差异

## 七、颜色命名建议

不要用 `.red`、`.blue`、`.green` 这类裸颜色名。

优先使用语义命名：

- `bg-base`
- `bg-elevated`
- `panel-fill`
- `panel-edge`
- `accent-main`
- `accent-danger`
- `accent-reward`
- `text-primary`
- `text-secondary`
- `text-inverse`
- `glow-main`
- `shadow-projection`

## 八、最后判断一次有没有落回默认样子

出稿前自问：

1. 如果把颜色去掉，这套页面骨架还认得出题材吗？
2. 如果只看按钮、卡片、标题条，它们是不是来自同一个世界观？
3. 如果把标题换掉，这套 UI 会不会像另一个常见手游页面直接复用？
4. 有没有哪一页明显退回“通用后台 / 通用蓝金 / 通用玻璃赛博”？

只要这四个问题里有一个答案是“会”，就说明风格信息还不够，不能直接开写最终页面。
