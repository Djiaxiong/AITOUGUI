# AIToUGUI 主题与基础元素编写规范

## 目标

这份文档说明两件事：

- `ThemeDefinition` 负责整站视觉语言
- `ElementLibrary` 负责语义元素到 Unity 控件类型的默认映射

页面自身只负责结构和局部布局，不负责重新定义整套视觉规则。

## ThemeDefinition 当前建议维护的字段

### 颜色

- `pageBackground`
- `panelFill`
- `cardFill`
- `buttonFill`
- `accentColor`
- `textPrimary`
- `textSecondary`
- `outlineColor`
- `shadowColor`
- `glowColor`

### 尺寸

- `pagePadding`
- `panelRadius`
- `cardRadius`
- `buttonRadius`
- `outlineWidth`
- `shadowSize`
- `shadowBlur`
- `glowBlur`
- `glowIntensity`

### Token

推荐把以下信息写成 token：

- 页面背景
- 主面板颜色
- 卡片颜色
- 强调色
- 圆角梯度
- gap 系列
- line 系列

## 形态语言合同

不要再把“视觉”理解成只有颜色和阴影。

当前推荐把一个基础元素拆成四层：

- `data-ui-element`
- `data-ui-variant`
- `data-ui-shape`
- `data-ui-frame`

其中：

- `element`
  - 负责语义控件类型和默认 Unity 控件映射
- `variant`
  - 负责项目元素库允许的同类变体
- `shape`
  - 负责轮廓家族
- `frame`
  - 负责边框和线条语言

### 当前推荐 shape 集合

- `roundrect`
- `per-corner`
- `capsule`
- `cut-corner`
- `plate`
- `banner`

### 当前推荐 frame 集合

- `solid`
- `outline`
- `hairline`
- `glow`

## 当前推荐 role 集合

至少覆盖：

- `window/main`
- `window/overlay`
- `panel/primary`
- `panel/dark`
- `card/resource`
- `card/info`
- `button/primary`
- `button/secondary`
- `button/danger`
- `button/ghost`
- `chip/tag`
- `text/title`
- `text/muted`
- `text/title-inverse`
- `text/inverse-muted`

## Visual Preset 建议

当前最少准备三类：

- `panel/default`
- `card/default`
- `button/default`

每个 preset 至少明确：

- 是否填充
- 主填充色
- 是否渐变
- 圆角
- 描边宽度和颜色
- 阴影
- 发光

## Motion Preset 建议

当前最少准备三类：

- `motion/default`
- `motion/page`
- `motion/button`

建议：

- 页面进入用轻量 `Fade` 或轻位移
- 按钮 hover 用轻抬升
- 按钮 press 用轻缩放
- 不要用大幅 bounce

## ElementLibrary 默认映射建议

导入站点时至少要能映射这些元素：

- `window/main`
- `window/overlay`
- `panel/primary`
- `panel/dark`
- `card/resource`
- `card/info`
- `button/primary`
- `button/secondary`
- `button/danger`
- `button/ghost`
- `chip/tag`

## 每个元素映射项至少包含

- `elementId`
- `controlType`
- `defaultRole`
- `visualPresetId`
- `motionPresetId`
- `defaultSize`

## 关于 `defaultSize`

这里是当前很重要的边界。

`defaultSize` 的意义是：

- 给元素映射一个标准尺寸参考
- 供明确 opt-in 的节点使用

它默认不是所有节点的最终尺寸。

当前规则是：

- HTML 页面里关键控件仍然要写显式宽高
- 只有写了 `data-ui-template-size="true"`，默认尺寸才允许接管

这里的 `data-ui-template-size` 是默认尺寸 opt-in 合同，不是鼓励页面按模板复用。

## 推荐用法

### role 负责视觉语义

```html
<div data-ui-role="panel/primary"></div>
```

### element 负责默认控件映射

```html
<button
  data-ui-element="button/primary"
  data-ui-shape="per-corner"
  data-ui-frame="outline"
  data-ui-role="button/primary"
  class="slab-button shape-per-corner frame-outline"
  style="width:414px; height:78px;">
  Start
</button>
```

说明：

- `data-ui-role` 是 CSS 和视觉语义入口
- `data-ui-element` 是给 Unity 元数据和控件映射用的
- `data-ui-shape` 是给 Unity 轮廓后端和作者态风格分叉用的
- `data-ui-frame` 是给 Unity 线条语言和作者态风格分叉用的
- 最终尺寸仍建议由 HTML 显式给出

## 当前协作建议

1. 先定 `LayoutArchetype / ShapeLanguage / FrameLanguage / OrnamentLanguage`
2. 再定主题 token 和 role 视觉
3. 再定 ElementLibrary 的最小映射集合
4. 再生成整站 HTML 页面
5. 最后导入 Unity 烘焙 prefab
